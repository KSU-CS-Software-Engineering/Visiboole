How To Read

@Name represents a required attribute named 'Name'.
@Name represents an optional attribute named 'Name'.
[ represents a required '['.
library represents a required 'library'.

Variables

Description: Variables are used to name the existance of a network of virtual wires and gates. Variables can either be independent or dependent. Independent variables are variables that don't depend on any variable for its value. Dependent variables are variables that depend on another variable for its value.
Valid Syntaxes:
 @Namespace@Bit
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  @Bit represents the specific index or bit of a namespace. This identifies a single value in that variable's namespace.
  Examples:
   test
   a0
   b31

 @Namespace[@Bit]
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  @Bit represents the specific index or bit of a namespace. This identifies a single value in that variable's namespace.
  Examples:
   a[0]
   b[31]

 @Namespace[@LeftBound.@Step.@RightBound]
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  @LeftBound represents the first index or bit you want to reference in that variable's namespace.
  @Step represents the difference between each index or bit. This value must not be negative. If no value is provided, the value will be 1.
  @RightBound represents the last index or bit you want to reference in that variable's namespace.
  Examples:
   a[3..0]
   b[0.1.4]
   c[2.3.6]
  Note: There is an enforced 32 bit limit. Bounds must be between 0 and 31 and the step must be between 1 and 31.

 @Namespace[]
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  Examples: a[], b[]
  Note: In order to use this syntax, you must have specified an explicit dimension of the namespace somewhere. This will reference all bits from the highest bit to the lowest bit ever used.

Constants

Description: Constants are used to represent positive numbers for a variable value.
Valid Syntaxes:
 @BitCount'@Format@Value
  @BitCount represents the number of bits you want the value to be.
  @Format represents the format of the constant. Valid formats are d for decimal, h for hex and b for binary.
  @Value represents the value of the constant. This value must be the same format as the format chosen.
  Examples:
   'd1
   4'd1
   'b1001
   'hFFFF
  Note: There is an enforced 32 bit limit. The bit count can't exceed 31 and the value must be 32 bits or less.

 @Value
  @Value represents a decimal value of a constant.
  Examples:
   3
   12
   37
  Note: There is an enforced 32 bit limit. This value can't exceed 4294967296.

 @BitCount'@Value
  @BitCount represents the number of bits for the specified decimal value.
  @Value represents a decimal value of a constant.
  Examples:
   4'2
   1'1
   8'37
  Note: There is an enforced 32 bit limit. Bit count can't exceed 32 and value can't exceed 4294967296.

Concatenations

Description: Concatenations are used to group variables and/or constants into a single set of values.
Syntax:
 {@List}
  @List represents a list of variables or constants seperated by spaces.
  Examples:
   {a b c}
   { a[3..0] a }
   { a3 a2 a1 a0 }
   {3'd4 a}
  Note: Constants in these concatenation fields must specify a bit count.

Operators

Description: Operators are used as gates that combine value(s) into new value(s).
Operators:
 Not Operator: ~
  Not operator takes the value of a parentheses set, concatenation field or a variable and negates it.
  Examples:
   ~a
   ~a3
   ~(a)
   ~{a}
  Note: This operator must be attached to an open parenthesis, beginning of a concatenation field or variable.

 And Operator: (Space)
  And operator takes the value on the left and performs a bitwise and with the value on the right.
  Examples:
   a b
   a3 b3
   (a3) (b3)
   a 0
   (a3)(b3)
  Note: When there is no operator between two variables or a variable and a parenthesis, this is assumed to be an and operation.

 Or Operator: |
  Or operator takes the value on the left and performs a bitwise or with the value on the right.
  Examples:
   a | b
   a3 | b3
   (a3) | (b3)
   a|0
   (a3)|(b3)

 Xor Operator: ^
  Xor operator takes the value on the left and performs a bitwise exlcusive or with the value on the right.
  Examples:
   (a ^ b)
   (a3 ^ b3)
   ((a3) ^ (b3) ^ (c3))
   (a^0)
   ((a3)^(b3))
  Note: Xor operations must be in parentheses and must be the only operators in those parentheses.

 Equal To Operator: ==
  Equal to operator takes the value on the left and right an evaluates to true if both values are the same.
  Examples:
   (a == b)
   (a3 == b3)
   ((a3) == (b3) == (c3))
   (a==0)
   ((a3)==(b3))
  Note: Equal to operations must be in parentheses and must be the only operators in those parentheses.

 Addition Operator: +
  Addition operator takes the value on the left and adds it to the value on the right.
  Examples:
   a + b
   a + 1
   a + 5
  Note: Addition operations can not be used with boolean operators. They can only be used with subtraction operators.

 Subtraction Operator: -
  Subtraction operator takes the value on the left and subtracts it from the value on the right.
  Examples:
   a - b
   a - 1
   a - 5
  Note: Subtraction operations can not be used with boolean operators. They can only be used with addition operators.

Display Statements

Description: Display Statements are used to toggle independent variable values or display variable values in new formats.
Syntax:
 @List;
  @List represents a list of variables, constants or formatted fields seperated by spaces.
  Examples:
   a;
   a3 a2 a1 a0;
   a[3..0];
   %u{a[]} a[3..0];
   %b{a} b c %b{d};

Assignment Statements

Description: Assignment Statements are used to assign values to your variables.
Syntax:
 @SingleVarOrConcat @Assignment @Expression;
  @SingleVarOrConcat represents a variable or a concatenation of variables you want to assign a value to.
  @Assignment represents the assignment operator. (<= or =)
  @Expression represents the expression that will be evaluted and assigned to the variables or concatenation on the left side of the assignment operator.
Examples:
 a = b;
 a[3..0] = ~(b[3..0] | c[3..0]);
 {a3 a2 a1 a0} = b[3..0] c[3..0] | d[3..0] e[3..0];
 a[] <= a[] + 1;
Note: <= represents a sequential assignment that will assign the value of the right side to the left side on a clock tick.

Header Statements

Description: Header Statements are used to define a design module that can be used in other designs.
Syntax:
 @DesignName(@List:@List);
  @DesignName represents the name of the current design.
  @List represents a list of variables or concatenations of variables seperated by commas.
  Examples:
   design(a,b,c:d);
   test(a[3..0], b[3..0] : c[3..0], d[3..0]);
   design2({a3 a2 a1 a0} : b[3..0]);
  Note: Variables on the left side of the ':' are input variables of the design and must be independent variables. Variables on the right side of the ':' are output variables of the design. All vectors in the header, must have explicit dimensions.

Instantiation Statements

Description: Instantiation Statements are used to create an instance of a design module.
Syntax:
 @DesignName.@InstantName(@List:@List);
  @DesignName represents the name of the design you are instantiating.
  @InstantName represents the name of your instantiation of that design.
  @List represents a list of variables or concatenations of variables seperated by commas.
  Examples:
   design.d1(a,b,c:d);
   test.t1(a[3..0], 4'4 : b[3..0]);
   design2.d1({a3 a2 a1 a0} : {NC b[2..0]});
  Note: Variables/constants on the left side of the ':' are input values of the instantiation. Variables on the right side of the ':' are output values of the design. NC as an output means "Not Connected". This is to be used when you don't want to save the value but want to see the value.

Library Statements

Description: Library Statements are used to include additional folders to scan for module designs.
Syntax: #library @LibraryName;
 @LibraryName represents the name of the folder you want to add for design lookups.
Examples:
 #library ..;
 #library modules;
 #library ..\others;
 #library \C:\Users\user\Desktop;
Note: Library names are assumed to be relative paths. You can use relative path syntax to go up directories. If you want an absolute path, place a '\' before the path.
