How To Read

@Name represents a required attribute named 'Name'.
@Name represents an optional attribute named 'Name'.
[ represents a required '['.
library represents a required 'library'.

Variables

Description: Variables are used to name the existance of a network of virtual wires and gates.
Valid Syntaxes:
 @Namespace@Bit
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  @Bit represents the specific index or bit of a namespace. This identifies a single value in that variable's namespace.
  Examples:
   test
   a0
   b31

 @Namespace[@Bit]
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  @Bit represents the specific index or bit of a namespace. This identifies a single value in that variable's namespace.
  Examples:
   a[0]
   b[31]

 @Namespace[@LeftBound.@Step.@RightBound]
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  @LeftBound represents the first index or bit you want to reference in that variable's namespace.
  @Step represents the difference between each index or bit. This value must not be negative. If no value is provided, the value will be 1.
  @RightBound represents the last index or bit you want to reference in that variable's namespace.
  Examples:
   a[3..0]
   b[0.1.4]
   c[2.3.6]
  Note: There is an enforced 32 bit limit. Bounds must be between 0 and 31 and the step must be between 1 and 31.

 @Namespace[]
  @Namespace represents any combination of alphanumeric characters that doesn't end in a number. This identifies the variable.
  Examples: a[], b[]
  Note: In order to use this syntax, you must have specified an explicit dimension of the namespace somewhere. This will reference all bits from the highest bit to the lowest bit ever used.

Constants

Description: Constants are used to represent positive numbers for a variable value.
Valid Syntaxes:
 @BitCount'@Format@Value
  @BitCount represents the number of bits you want the value to be.
  @Format represents the format of the constant. Valid formats are d for decimal, h for hex and b for binary.
  @Value represents the value of the constant. This value must be the same format as the format chosen.
  Examples:
   'd1
   4'd1
   'b1001
   'hFFFF
  Note: There is an enforced 32 bit limit. The bit count can't exceed 31 and the value must be 32 bits or less.

 @Value
  @Value represents a decimal value of a constant.
  Examples:
   3
   12
   37
  Note: There is an enforced 32 bit limit. This value can't exceed 4294967296.

 @BitCount'@Value
  @BitCount represents the number of bits for the specified decimal value.
  @Value represents a decimal value of a constant.
  Examples:
   4'2
   1'1
   8'37
  Note: There is an enforced 32 bit limit. Bit count can't exceed 32 and value can't exceed 4294967296.

Concatenations

Description: Concatenations are used to group variables and/or constants into a single set of values.
Syntax:
 {@List}
  @List represents a list of variables or constants seperated by spaces.
  Examples:
   {a b c}
   { a[3..0] a }
   { a3 a2 a1 a0 }
   {3'd4 a}
  Note: Constants in these concatenation fields must specify a bit count.

Operators

Description: Operators are used as gates that combine value(s) into new value(s).
Operators:
 Not Operator: ~
  Not operator takes the value of a parentheses set, concatenation field or a variable and negates it.
  Examples:
   ~a
   ~a3
   ~(a)
   ~{a}
  Note: This operator must be attached to an open parenthesis, beginning of a concatenation field or variable.

 And Operator: (Space)
  And operator takes the value on the left and performs a bitwise and with the value on the right.
  Examples:
   a b
   a3 b3
   (a3) (b3)
   a 0
   (a3)(b3)
  Note: When there is no operator between two variables or a variable and a parenthesis, this is assumed to be an and operation.

Header Statements

Description: Header Statements are used to define a design module that can be used in other designs.
Syntax:
 @DesignName(@List:@List);
  @DesignName represents the name of the current design.
  @List represents a list of variables or concatenations of variables seperated by commas.
  Examples:
   design(a,b,c:d);
   test(a[3..0], b[3..0] : c[3..0], d[3..0]);
   design2({a3 a2 a1 a0} : b[3..0]);
  Note: Variables on the left side of the ':' are input variables of the design and must be independent variables. Variables on the right side of the ':' are output variables of the design.

Instantiation Statements

Description: Instantiation Statements are used to create an instance of a design module.
Syntax:
 @DesignName.@InstantName(@List:@List);
  @DesignName represents the name of the design you are instantiating.
  @InstantName represents the name of your instantiation of that design.
  @List represents a list of variables or concatenations of variables seperated by commas.
  Examples:
   design.d1(a,b,c:d);
   test.t1(a[3..0], 4'4 : b[3..0]);
   design2.d1({a3 a2 a1 a0} : {NC b[2..0]});
  Note: Variables/constants on the left side of the ':' are input values of the instantiation. Variables on the right side of the ':' are output values of the design. NC as an output means "Not Connected". This is to be used when you don't want to save the value but want to see the value.
